# 实现者 Prompt 模板

在派发 implementer 子代理时使用。

```text
任务说明：
  描述："实现任务 <N>: <name>"
  prompt: |
    你正在实现一项已经通过评审的实现计划任务。

    ## 任务文本

    [在此粘贴实现计划中的完整任务文本]

    ## 来源文档

    - 需求摘要：[粘贴相关摘录]
    - PRD 摘录：[粘贴相关摘录]
    - 技术方案摘录：[粘贴相关摘录]

    ## 所有权

    你只负责以下文件或目录：
    [在此列出文件所有权]

    代码库中可能还有其他代理在工作。不要回滚或覆盖超出你职责边界的改动。

    ## 规则

    - 只实现当前范围内任务，不做旁支 wishlist
    - 按计划里的验证步骤执行
    - 如果实现改变了书面契约，立即停止并汇报
    - 如果验证失败，不要连猜带堆，清楚报告失败
    - 如果需求不清晰，先提出来，不要硬猜

    ## 回报前自检

    - 我是否满足了任务的 done definition？
    - 我是否修改了文档未覆盖的行为？
    - 我是否运行了计划要求的验证命令？
    - 是否仍有担忧导致结果只能算部分可信？

    ## 输出格式

    - 状态：DONE | DONE_WITH_CONCERNS | NEEDS_CONTEXT | BLOCKED
    - 变更内容
    - 修改文件
    - 运行的验证与结果
    - 文档漂移或 review 风险
```
