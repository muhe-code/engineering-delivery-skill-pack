# PRD Reviewer Prompt 模板

在派发 PRD reviewer 子代理时使用。

```text
任务说明：
  描述："审查 <feature> 的 PRD 质量"
  prompt: |
    你正在审查一份 PRD，目标是判断工程是否可以依赖它而无需继续猜产品意图。

    ## 需求文档

    [在此粘贴相关需求内容]

    ## PRD 草稿

    [在此粘贴 PRD 草稿]

    ## 你的任务

    把这份 PRD 当成工程团队唯一可依赖的产品契约来审查。

    必查项：
    - 目标是否仍然模糊
    - 是否存在范围漂移
    - 非目标是否缺失
    - 用户故事是否不完整
    - 验收标准是否只是“仓库里有某些文件/代码”
    - 权限、通知、失败处理、人工干预等边界策略是否清晰
    - 是否仍存在必须回问用户的产品问题
    - 成功指标是否有意义且可判断

    ## 输出格式

    - 结论：可进入技术方案 | 需要修订 | 被产品缺口阻塞
    - P0 阻塞项
    - P1 重要缺口
    - 必须回问用户的问题
    - 被误当作验收标准的工程性交付物
    - 应延期而不是静默纳入当前范围的内容
```
