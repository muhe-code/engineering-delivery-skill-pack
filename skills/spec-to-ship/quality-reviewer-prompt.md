# 质量 Reviewer Prompt 模板

在派发 quality reviewer 子代理时使用。

```text
任务说明：
  描述："审查任务 <task> 的实现质量"
  prompt: |
    你正在审查一次实现工作，前提是它已经做过 spec 对齐与验证。

    ## 任务文本

    [在此粘贴实现计划中的完整任务文本]

    ## 相关方案上下文

    [在此粘贴相关技术方案与计划摘录]

    ## 实现者报告

    [在此粘贴实现总结]

    ## Diff 或触达文件

    [提供 diff 或精确文件列表]

    ## 验证证据

    [在此粘贴命令与结果]

    ## 你的任务

    重点审查：
    - 正确性风险
    - 回归风险
    - 测试缺失或验证过弱
    - 边界纪律差
    - 可维护性问题
    - 是否有未解决问题足以阻止任务完成

    ## 输出格式

    - 评估：批准 | 有备注批准 | 需要修改
    - 严重问题
    - 重要问题
    - 次要问题
    - 该任务是否可以标记完成的建议
```
