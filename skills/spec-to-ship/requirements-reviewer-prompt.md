# 需求文档 Reviewer Prompt 模板

在派发 requirements reviewer 子代理时使用。

```text
任务说明：
  描述："审查 <feature> 的需求文档质量"
  prompt: |
    你正在审查一份需求文档，目标不是润色文字，而是判断它是否允许进入 PRD 阶段。

    ## 用户请求摘要

    [在此粘贴当前请求摘要]

    ## 仓库上下文

    [在此粘贴相关文件、约定、现有行为或约束]

    ## 需求文档草稿

    [在此粘贴需求草稿]

    ## 你的任务

    独立审查这份需求文档。不要在未被要求时整篇重写。重点找出会导致后续 PRD、技术方案或实现漂移的问题。

    必查项：
    - 是否存在被误写成需求的方案假设
    - 是否缺少角色、核心场景、权限边界、风险边界、失败策略
    - 是否缺少用户可感知的验收信号
    - 是否仍有必须由用户确认的 P0 问题
    - 是否仍需要工程替用户做产品决策

    ## 输出格式

    - 结论：可进入PRD | 需要修订 | 被产品缺口阻塞
    - P0 阻塞项
    - P1 重要缺口
    - 必须回问用户的问题
    - 被误写成需求的方案假设
    - 可以保留的强项
```
