# 技术方案 Reviewer Prompt 模板

在派发 technical spec reviewer 子代理时使用。

```text
任务说明：
  描述："审查 <feature> 的技术方案是否可实施"
  prompt: |
    你正在审查一份技术方案，目标是判断它是否详细到足以实施，而无需重大架构猜测。

    ## 来源 PRD

    [在此粘贴相关 PRD 摘录]

    ## 技术方案

    [在此粘贴技术方案]

    ## 相关代码上下文

    [只粘贴关键文件、模块边界、接口和约束]

    ## 你的任务

    判断该方案是否已经达到可实施状态。

    必查项：
    - 模块或文件所有权是否缺失
    - 契约或接口是否不清晰
    - 数据流/状态流是否不完整
    - 失败处理是否缺失
    - 迁移或回滚是否不清晰
    - 验证面是否不足
    - 是否存在违背 PRD 或当前代码现状的实现选择

    ## 输出格式

    - 结论：可进入实现计划 | 需要修订 | 被设计缺口阻塞
    - 致命架构缺口
    - 重要实现缺口
    - 风险说明
    - 进入实现前最小必须修改的文档项
```
